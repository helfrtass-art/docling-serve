=============================================================================
SECURITY IMPROVEMENTS
=============================================================================

✅ PASSWORD HASHING
   - Implementation: bcrypt with salt rounds=12 (NIST approved)
   - Hash format: $2b$12$... (60 characters)
   - Fallback: Plaintext if bcrypt unavailable (logs warning)
   - Protection: Rainbow table attacks infeasible

✅ COOKIE SECURITY
   - Added flags: Secure, HttpOnly, SameSite=Strict
   - Secure: Only sent over HTTPS
   - HttpOnly: Not accessible to JavaScript (prevents XSS)
   - SameSite=Strict: Prevents CSRF attacks

✅ FILE PERMISSIONS
   - users.json created with mode 0600 (owner read/write only)
   - Protects against unauthorized local access
   - Set via os.chmod() after JSON dump

✅ NO HARDCODED CREDENTIALS
   - Removed: Default user from source code
   - Removed: Hardcoded password values
   - Implemented: Environment variable initialization
   - Added: init_admin.py for secure setup

✅ SECRET KEY MANAGEMENT
   - Requires: DOCLING_SECRET_KEY environment variable
   - Recommended: Use OpenShift Secrets for injection
   - Rotation: Invalidates all sessions on change
